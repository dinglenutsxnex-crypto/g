## Rule
`Godot.PackedVector3Array`, `Godot.PackedVector2Array`, `Godot.PackedFloat32Array`, `Godot.PackedInt32Array` do NOT exist in the NuGet `GodotSharp` 4.4.0 package distributed via nuget.org. The package provides only a stripped set of bindings; the full C# API including packed arrays is generated by the Godot editor.

**Stubs:** `scripts/firstpass/GodotPackedArrayStubs.cs` — declares these four types in the `Godot` namespace as `List<T>` wrappers with `.Count`, `.Resize(n)`, `[i]` indexer, and `.Add(v)`.

**Affected file:** `scripts/SkinMeshCombineUtility.cs` uses these types for mesh-combining. The explicit `Godot.` prefix is used there (not bare names).

**Why:** If Godot editor is ever installed in this environment, the generated GodotSharp.dll will take precedence. Our stubs only compile-stub the types; mesh-combination behaviour will need further review at that point.
